# 安全与数据处理

## 默认策略

公开版默认使用 `move_after_verified_upload`：只有上传成功且云端同名同大小验证通过，才把本地视频移动到 `recordings\uploaded`。

用户也可以选择 `keep_local`。公开版不提供自动永久删除。旧配置中的删除模式会安全降级为移动。

## 移动本地文件的必要条件

云端模式下必须同时满足：

1. 本地文件已停止写入并可以独占读取。
2. rclone 上传退出码为 0。
3. rclone 能读取目标远程目录。
4. 云端存在名称完全相同的文件。
5. 云端文件大小与本地文件大小一致。
6. 本地文件位于 ScreenAgent 的 `recordings\raw` 目录内。
7. 本地文件不是目录、符号链接或重解析点。
8. 本地目标使用唯一名称，不覆盖已有文件。

任何一项不满足，本地视频都必须保留在原位。

## 凭据处理

- 密码使用 PowerShell `SecureString` 输入。
- 明文仅在当前进程内短暂存在，并通过标准输入发送给 `rclone obscure -`。
- 明文密码不会放进 rclone 的命令行参数、ScreenAgent 配置、日志或安装报告。
- 写入 rclone 配置的是混淆值；该值可由 rclone 还原，不等同于强加密。
- ScreenAgent 使用独占的 `%USERPROFILE%\ScreenAgent\config\rclone.conf`，不会写入用户的全局 rclone 配置。
- 配置文件创建后会关闭继承权限，只允许当前 Windows 用户访问。
- 通用 WebDAV 必须使用 HTTPS，避免凭据和录像通过明文 HTTP 传输。
- rclone 配置必须保持私密，不能提交到 Git、附在 Issue 中或发送给陌生人。

## 卸载

卸载器只移除计划任务、快捷方式，并在明确输入 `DELETEAPP` 时删除程序脚本目录。它不会递归删除录屏、日志、会话或配置。用户如需清理数据，必须在资源管理器中自行检查后处理。

## 验证边界

WebDAV 不一定提供内容哈希，因此同名同大小只能证明远程对象与本地文件具有相同名称和长度，不能提供逐字节完整性证明。重要视频建议使用 `keep_local` 或额外备份。
