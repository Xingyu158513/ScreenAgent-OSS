# ScreenAgent 故障排查

## OBS 未安装或找不到

现象：启动录制时提示找不到 `obs64.exe`。

处理：从 https://obsproject.com/download 安装 OBS，然后重新运行安装向导。常见路径：

```text
C:\Program Files\obs-studio\bin\64bit\obs64.exe
```

## OBS 预览黑屏

1. 用管理员身份启动 OBS。
2. 删除后重新添加“显示器采集”。
3. 右键采集源，选择“变换 → 适应屏幕”。
4. 在 Windows 图形设置中，把 OBS 设为高性能。
5. 笔记本双显卡环境中，尝试让 OBS 与被采集应用使用同一显卡。

## 找不到 rclone

从 https://rclone.org/downloads/ 下载 Windows 版本，解压后将 `rclone.exe` 放到：

```text
C:\Tools\rclone\rclone.exe
```

然后重新运行 `install.bat`。

## 坚果云出现 401 Unauthorized

不要使用坚果云登录密码。请在坚果云网页的第三方应用管理中重新生成应用密码，再重新安装或运行配置向导。

帮助：https://help.jianguoyun.com/?p=2064

## 上传成功但验证失败

先查看日志：

```powershell
Get-Content "$env:USERPROFILE\ScreenAgent\logs\report.txt" -Tail 50
```

再读取配置并检查云端：

```powershell
$c = Get-Content "$env:USERPROFILE\ScreenAgent\config\config.json" -Raw | ConvertFrom-Json
& $c.rclone_exe lsjson $c.remote_root --files-only
```

验证失败时本地文件会留在 raw，不要手动删除。常见原因是网络延迟、权限不足、远程目录错误或云端文件大小尚未刷新。

## 后台任务没有运行

```powershell
Get-ScheduledTask -TaskName "ScreenAgent-AutoUpload"
Start-ScheduledTask -TaskName "ScreenAgent-AutoUpload"
```

如果任务不存在，可手动运行：

```powershell
& "$env:USERPROFILE\ScreenAgent\app\auto_archive.ps1"
```

## 开机弹出 PowerShell 窗口

确认计划任务执行的是 `wscript.exe`，参数指向：

```text
%USERPROFILE%\ScreenAgent\app\run_auto_archive_hidden.vbs
```

## raw 中有旧视频

旧文件可能是上传失败、验证失败或超过 6 小时的恢复文件。先查日志和云端，不要直接删除。后台程序会把过期会话的视频归入 `recovered/unknown`。

## 收集诊断信息

以下命令不会修改数据：

```powershell
Get-ScheduledTask -TaskName "ScreenAgent-AutoUpload" | Format-List *
Get-Content "$env:USERPROFILE\ScreenAgent\config\config.json" -Raw
Get-Content "$env:USERPROFILE\ScreenAgent\logs\report.txt" -Tail 100
Get-ChildItem "$env:USERPROFILE\ScreenAgent\recordings\raw" -File
```

分享配置前检查并移除个人账号、服务器地址和个人目录信息。不要分享应用密码或 rclone 配置文件。
